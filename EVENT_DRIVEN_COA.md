# MissionOrch - 基于事件和条件的COA矩阵说明

## 概述
MissionOrch系统已经从基于固定时间的COA矩阵转换为基于事件和条件的矩阵。这种转变更符合实际军事行动的动态性和不确定性。

## 传统时间驱动方法的问题
- 假设所有事件都按预定时间发生
- 无法处理行动中的不确定性
- 飞机位置无法仅凭时间确定（受速度、路线、天气等因素影响）

## 新的事件驱动方法
COA现在基于以下参数：

### 任务参数
- `start_condition`: 任务开始条件（如"到达目标区域"、"检测到威胁"）
- `end_condition`: 任务结束条件（如"威胁解除"、"燃料不足"）
- `location`: 任务执行位置
- `duration_min/max`: 最短/最长持续时间

### 示例
```
任务A: 
  - 开始条件: "F-35到达目标区域"
  - 结束条件: "威胁解除" 
  - 位置: "坐标X"
  - 类型: "DCA" (防御性制空)

任务B:
  - 开始条件: "威胁解除"
  - 结束条件: "返回基地"
  - 位置: "基地坐标"
  - 类型: "RTB" (返航)
```

## 优势
1. **适应性强**: 能够处理行动中的变化和不确定性
2. **真实性强**: 更贴近实际军事行动的动态特性
3. **仿真友好**: 仿真系统可以基于事件触发器动态执行任务
4. **灵活性高**: 无需预先设定精确时间表

## 仿真系统集成
仿真系统现在监听事件，而不是等待特定时间点：
- 事件监听器检测条件满足
- 动态触发相应任务
- 根据实时情况调整行动

这种方法使COA更具实用性和可操作性，同时为仿真系统提供了更灵活的执行框架。
